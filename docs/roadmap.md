# Roadmap

## Phase 1: Repository foundation

- Establish public documentation and contribution process.
- Add the skill template and metadata schema.
- Add lightweight validation and index generation scripts.
- Publish one complete sample skill.

## Phase 2: Early registry quality

- Add several high-quality skills across distinct categories.
- Improve tests and fixture coverage for registry scripts.
- Refine safety levels, risk tags, and compatibility metadata.
- Clarify maintainer review expectations.

## Phase 3: Registry maturity

- Add stronger schema validation if needed.
- Generate registry data consistently from skill metadata.
- Add compatibility examples for common agent runtimes.
- Define release and versioning conventions for skills.

## Not planned for the initial launch

- A website.
- A hosted API.
- Heavy framework dependencies.
- Runtime-specific packaging before the registry conventions are stable.
