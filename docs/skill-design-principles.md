# Skill Design Principles

## Reusable

A skill should solve a recurring problem for more than one user, task, or organization.

## Bounded

A skill should describe a clear capability with known inputs, outputs, and limits.

## Testable

A skill should include at least one basic test that checks expected behavior and required output properties.

## Honest about risk

A skill should name safety boundaries, permissions, risk tags, and failure modes directly.

## Compatible by evidence

Compatibility metadata should describe where the skill is expected to work and what assumptions it makes. Avoid unsupported claims.

## Agent-facing

`SKILL.md` should be written for an AI agent that needs to execute the workflow, not for a human reading a blog post.
