# Governance

Awesome Skills Library uses lightweight maintainer governance during the launch phase.

## Maintainers

Maintainers are responsible for reviewing contributions, preserving project scope, managing safety concerns, and keeping the repository structure coherent.

## Decision principles

Project decisions should favor:

- reusable agent capability packages over prompt collections;
- clear metadata and tests over informal descriptions;
- small maintainable scripts over heavy infrastructure;
- honest compatibility claims over broad unsupported claims;
- explicit safety boundaries over implicit assumptions.

## Contribution review

Maintainers may accept, request changes, or reject contributions. Rejection should include a clear reason, such as insufficient reusability, missing safety boundaries, weak tests, unclear ownership, or mismatch with project scope.

## Changes to governance

Governance may evolve as the contributor base grows. Material governance changes should be documented in this file and announced in the changelog.
